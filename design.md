产品需求文档 (PRD) - 灵感盲盒 (Inspiration Gacha)

1. 产品概述

灵感盲盒 是一款将“提示词工程”游戏化的 AI 辅助创作应用。用户通过类似二次元游戏的“抽卡”机制随机获得：【主体】、【风格】、【构图】、【光影】等要素卡片，系统自动将其组合并调用 LLM 进行润色，最终生成高质量的 AI 图像。

2. 核心价值

降低门槛：解决用户面对空白输入框时的“创作焦虑”。

趣味探索：将随机组合作为灵感来源，发现意想不到的视觉组合。

高质量输出：利用 LLM 将碎片化的词组转化为结构严谨、细节丰富的绘画指令。

3. 核心功能模块

3.1 抽卡系统 (The Gacha Engine)

卡池分类：

主体池 (Subject)：人物、生物、场景、抽象概念。

风格池 (Style)：赛博朋克、油画、吉卜力风、Ukiyo-e、虚幻引擎渲染等。

动作/姿态池 (Pose)：奔跑、战斗姿态、冥想（后期可扩展 ControlNet）。

细节补充池 (Details)：特定光影、季节、材质、装饰品。

抽卡逻辑：

用户选择“十连抽”或“单抽”。

系统从各个维度的卡池中随机抓取关键词。

稀有度设定：普通词（白）、高级词（蓝）、史诗词（紫）、传说级艺术风格（金）。

3.2 灵感合成与 LLM 润色

关键词槽位：用户可以从抽到的卡片中筛选出 3-5 张放入“合成槽”。

智能润色 (The Polisher)：

调用 Gemini API，将抽中的碎片关键词（如：女孩，赛博朋克，雨夜，霓虹）扩充为一段描述性的、符合 Stable Diffusion 或 Midjourney 逻辑的长 Prompt。

支持用户设定“润色强度”（从“忠于原词”到“AI 自由发挥”）。

3.3 图像生成集成

预览功能：润色完成后，直接调用图像生成模型（如 Imagen 4.0）展示效果。

参数控制：支持选择长宽比（1:1, 16:9, 9:16）。

3.4 进阶控制（后期规划）

背景锁定：固定背景卡片，只更换主体。

姿态参考 (ControlNet)：抽到特定的“动作卡”时，后端自动挂载对应的 OpenPose 模型。

卡片收藏夹：用户可以保存自己喜欢的“金色词条”以便重复使用。

4. 用户路径 (User Journey)

进入应用：展示当前的“今日特选卡池”（如：东方幻想、未来工业）。

点击抽卡：触发抽卡动画，展示抽中的卡片及其视觉预览。

筛选组合：用户点击选中的卡片，卡片飞入合成框。

智能润色：点击“注入灵魂”按钮，LLM 生成一段优美的描述性文字。

生成图像：点击生成，等待图像渲染。

保存与分享：保存图片及对应的“提示词配方”。

5. UI/UX 设计风格建议

视觉风格：建议采用“暗黑模式 + 霓虹发光”或“极简艺术风”。

动画效果：

卡片翻转时的闪光特效（对应不同稀有度）。

合成时的粒子聚合效果。

交互细节：

长按卡片可以查看该关键词的详细定义和示例图。

拖拽式操作，增强“组合”的实感。

6. 技术架构初步构想

前端：React / Next.js (用于快速构建交互丰富的 UI)。

逻辑层：

Randomizer: 处理基于概率的抽卡逻辑。

LLM Wrapper: 封装对 Gemini 2.5 Flash 的调用，负责 Prompt 的转换和增强。

后端/存储：

Firestore: 存储用户的卡片库、生成记录和图片 URL。

Image Gen API: 集成 Google Imagen 或 SD API。

7. 商业化方案（参考）

体力制：每日赠送有限的抽卡次数。

限定卡池：与特定艺术家或节日联名的特殊词条卡池。

配方分享：用户分享自己生成的“神仙配方”，获得奖励。
